## Purpose

Prove the Read-Only Memory Auditor tracks memory health across time without:
- lifecycle hooks
- memory mutation
- retrieval mutation
- automatic scheduling

## Trial runs (minimum 5)

| Run | Suggested trigger context |
|-----|---------------------------|
| run-001 | baseline after merge |
| run-002 | after ordinary session |
| run-003 | after session reflection |
| run-004 | after new self-memory or belief candidate |
| run-005 | after creator permission clarification |

**Do not fabricate stable-memory events** only to feed the auditor.

## Per-run command

```bash
bash scripts/manual_shadow_audit.sh run-001 "baseline after merge"
```

Equivalent:

```bash
python3 -m memory_auditor runtime-sanitized \
  --db state/dioo.db \
  --fixture memory_auditor/fixtures/rt04.json \
  --run-id run-001 \
  --trigger-context "baseline after merge" \
  -o audit-reports/manual-shadow-run-001.json
```

Reports contain **aggregate metrics only** — no raw memory content.

## Compare across runs

After two or more runs:

```bash
python3 -m memory_auditor compare-runs \
  audit-reports/manual-shadow-run-001.json \
  audit-reports/manual-shadow-run-002.json \
  -o audit-reports/shadow-comparison.json
```

Comparison outputs (read-only):
- record growth by type
- lineage coverage delta vs baseline
- new orphan references
- permission / identity / summary-authority risk changes
- semantic pattern changes
- adapter error changes
- stop-condition violations

**Record count increases are not automatically degradation** — inspect `records_by_type` deltas.

## Metrics tracked per run

- `records_scanned`, `records_by_type`
- `lineage_status`, `lineage_edges_evaluated`, `lineage_resolution_rate`
- `derived_records_total`, `derived_records_with_source_refs`, `derived_records_missing_source_refs`
- `lineage_coverage_rate`, `lineage_coverage_by_type`
- `orphan_source_reference_count`
- `summary_authority_rate`
- `permission_integrity_findings`, `identity_findings`
- `runtime_only_patterns`
- `adapter_errors`, `snapshot_cleaned`, `mutations_performed`

Always report **both** `lineage_resolution_rate` (edge classification) and `lineage_coverage_rate` (record source refs).

## Success criteria (after 5 runs)

- `mutations_performed = 0` every run
- `snapshot_cleaned = deleted` every run
- `adapter_errors = []` or clearly explained
- `summary_authority_rate = 0`
- no fabricated lineage
- no new orphan refs without warning
- metrics never show `NOT_EVALUATED` as `1.0`
- deltas explainable by real events

## Stop conditions — halt trial immediately if

- `mutations_performed > 0`
- source integrity check abnormal
- snapshot cleanup failed
- raw memory in sanitized report
- auditor crash on real schema
- lineage metrics inconsistent
- auditor invoked by automatic lifecycle
